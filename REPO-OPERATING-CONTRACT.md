# REPO OPERATING CONTRACT - openclaw

## Owner / Purpose

- Owner: OpenClaw upstream product/runtime source, forked locally under `ljcairns/openclaw`.
- Purpose: core OpenClaw app/runtime, plugin/skill execution surfaces, provider integrations, local gateway behavior, and product docs.
- BRIU/OpenClaw system role: product/runtime dependency read by `openclaw-config`, `openclaw-mac`, and customer/reporting workflows.

## Truth Source

- Git truth: local `main` tracks `fork/main`; `origin` is upstream `openclaw/openclaw`.
- Operational BRIU/OpenClaw config truth does not live here. Use `~/repos/openclaw-config` for operator/customer workflow config, Discord/Gmail routing, scheduled customer reconciliation, and BRIU KV publishing.
- Cross-repo state truth lives in `~/repos/briu-website/docs/agent-os/state-map.json` and generated projections.

## CI / Deploy / Runtime

- CI policy: use repo workflows/package scripts for product/runtime changes; do not infer BRIU deployment state from this repo.
- Deploy policy: this repo does not directly deploy BRIU worker/admin surfaces or OpenClaw operator runtime config.
- Runtime policy: local/product runtime behavior may affect OpenClaw sessions, but Mac/VPS operator runtime config belongs to `openclaw-config` and `openclaw-mac`.

## State Domain Ownership

- Canonical cross-repo state-domain map lives in `~/repos/briu-website/docs/agent-os/state-map.json`.
- This repo does not own BRIU customer workflow state, dispatch terminal state, Gmail/Discord receipts, or operator secrets.
- It affects/reads these domains through product/runtime code and integration behavior:
  - `customer_ops`: customer workflow task execution can depend on OpenClaw runtime behavior, but OpenClaw config remains the writer.
  - `comms_artifacts`: provider messaging integrations can affect Gmail/Discord wrapper behavior; provider APIs and OpenClaw config receipts remain authoritative.
  - `credentials_and_runtime_config`: auth/gateway/provider semantics can change runtime expectations; secret locations and rotations remain outside this repo.
  - `scheduled_and_background_work`: product/runtime changes can affect scheduled OpenClaw tasks indirectly through wrappers/plugins.
- It reads but does not own:
  - `agent_context`: startup and task context compiled by BRIU/OpenClaw config surfaces.
  - `repo_topology`: repo membership and operating-contract inventory generated by BRIU Agent OS.
- Required checks after state-domain-adjacent changes: targeted product/runtime tests plus the affected `openclaw-config` or `openclaw-mac` smoke/audit when runtime config, provider wrappers, or task execution changes.
- Prose is receipt, not the register: customer/report/send/runtime-state claims must resolve through provider evidence, OpenClaw config maps, or BRIU Agent OS projections.

## Secrets / Artifacts

- Do not commit secrets, provider tokens, customer data, local keyrings, runtime env files, or generated provider receipts.
- Keep local media/cache/build output out of commits unless it is a deliberate product artifact tracked by this repo.
